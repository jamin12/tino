# 네임스페이스 타입 라벨 부여 기획

> Dooray: (TBD)
> 작성일: 2026-03-24

---

## 1. 배경

### 현재 상황

네임스페이스 생성 시 유형(USER / SYSTEM / CI_CD)을 지정할 수 있지만, 기존에 이미 존재하는 네임스페이스(kube-system, default, istio-system 등)에는 타입 라벨이 부여되어 있지 않다.

### 문제점

1. **미분류 네임스페이스 다수 존재**: 클러스터 초기 구성 시 생성된 시스템 NS, 수동 생성된 NS 등에 타입 라벨이 없음
2. **관리 사각지대**: 타입이 미지정이면 CCP 내부에서 분류·필터링·권한 관리 기준이 적용되지 않음
3. **개별 편집의 비효율**: 현재는 상세 → 편집(CCP-NSR-005)에서 Labels를 하나씩 수정해야 하므로, 다수의 NS에 동일 타입을 부여하려면 반복 작업 필요

---

## 2. 해결 방향

### 핵심 UX: "선택 → 일괄 부여"

리스트 화면(CCP-NS-001-L)에서 2단계로 끝나는 워크플로:

```
1. 선택: 전체 목록에서 체크박스로 대상 NS 다중 선택 (미지정 + 기존 타입 변경 모두 가능)
2. 부여: "라벨 부여" 버튼 → 모달에서 타입 선택 → 적용
```

- 타입 필터로 "미지정"만 빠르게 찾을 수도 있고, 전체 목록에서 타입 변경이 필요한 NS를 함께 선택할 수도 있음
- 이미 타입이 부여된 NS도 선택하면 새로운 타입으로 덮어쓰기 가능

### 변경 사항

| 대상 | 변경 내용 |
|------|----------|
| 리스트 화면 (CCP-NS-001-L) | 타입 컬럼 추가, 타입 필터 추가 |
| 신규 슬라이드 (CCP-NSR-006) | 일괄 라벨 부여 모달 화면 |

---

## 3. 화면 설계

### 3-1. 리스트 변경점 (CCP-NS-001-L 확장)

**타입 컬럼 추가:**

| 값 | Badge 스타일 | 설명 |
|----|-------------|------|
| USER | `blue-solid` | 사용자가 직접 생성/관리하는 NS |
| SYSTEM | `neutral` | CCP를 위한 리소스 관리 NS |
| CI_CD | `purple-solid` | CI/CD 파이프라인에서 사용하는 NS |
| 미지정 | `warning` | 타입 라벨이 없는 NS |

**타입 필터 추가:**
- FilterBar에 Select 추가: 전체 / USER / SYSTEM / CI_CD / 미지정

### 3-2. 라벨 부여 모달 (CCP-NSR-006)

**진입 경로:**
- 리스트에서 체크박스 선택 후 상단 "라벨 부여" 버튼 클릭

**모달 구성:**

```
┌─────────────────────────────────────────────┐
│ 🏷️  네임스페이스 타입 라벨 부여     N건    │
├─────────────────────────────────────────────┤
│                                             │
│ ℹ️ 타입 라벨은 CCP 내부에서 네임스페이스를  │
│    분류·관리하는 기준으로 사용됩니다.        │
│                                             │
│ ▎ 대상 네임스페이스                         │
│ ┌─────────────────────────────────────────┐ │
│ │ ● default          dev-cluster  미지정  │ │
│ │ ● kube-public      dev-cluster  미지정  │ │
│ │ ● istio-system     dev-cluster  미지정  │ │
│ │ ● sample-cicd      dev-cluster  CI_CD   │ │
│ ├─────────────────────────────────────────┤ │
│                                             │
│ ▎ 부여할 타입                               │
│ ┌─ [ USER ▼ ] ──────────────────────────┐  │
│                                             │
├─────────────────────────────────────────────┤
│                      [ 취소 ] [ N건 적용 ]  │
└─────────────────────────────────────────────┘
```

**동작:**
- Select로 타입 선택 (USER / SYSTEM / CI_CD)
- "적용" 버튼 → K8s API로 선택된 NS에 `ccp.cone-chain.com/type: {TYPE}` 라벨 일괄 추가
- 완료 후 리스트로 복귀, 타입 컬럼 갱신

---

## 4. 기술 상세

### 라벨 키

```
ccp.cone-chain.com/type: USER | SYSTEM | CI_CD
```

- K8s namespace Labels에 추가
- 기존에 해당 라벨이 있는 NS는 값이 덮어쓰기됨 (업데이트 가능)

### API 호출

각 대상 NS에 대해 `kubectl label namespace {name} ccp.cone-chain.com/type={TYPE} --overwrite` 와 동일한 PATCH 호출
