# CCP 디자인 시스템 구축

## 목표

CCP(CONE-Chain Pipeline) 대시보드 화면 설계서를 Tino 슬라이드로 구현할 때 사용할 **일관된 디자인 시스템**을 구축한다.
Figma에서 export된 모놀리식 코드(`example` 파일)에서 디자인 토큰과 재사용 컴포넌트를 추출하여 `content/ccp/_components/`에 체계적으로 정리한다.

## 배경

- Figma export 결과물(`example` 파일, ~2000줄)이 단일 파일에 모든 섹션이 하드코딩되어 있음
- 절대 좌표(`absolute top-[581px]`), 하드코딩 색상(`#0077ff`), 중복 코드(Pagination 2개, ContextMenu 2개) 등 유지보수 불가
- CCP 프로젝트에서 여러 화면(파이프라인 목록, 실행이력, 상세 등)을 슬라이드로 만들어야 하므로 재사용 가능한 컴포넌트가 필수

## 접근 방식

### 1. 디자인 토큰 정의 (`tokens.ts`)

example 파일에서 추출한 모든 디자인 값을 TypeScript 상수 객체로 정리한다.

| 토큰 카테고리 | 설명 |
|---------------|------|
| Colors | Primary(#0077ff), Success(#00b30e), Error(#da1e28), Warning(#dea600), Neutral 계열, 투명도 변형 |
| Typography | Pretendard 폰트 패밀리, 사이즈(10~24px), 웨이트(Regular~Bold), line-height, letter-spacing |
| Spacing | 패딩/갭 스케일 (2px~20px) |
| Shadows | 카드, 드롭다운, 툴팁, 패널 등 5종 |
| Borders | 색상, radius(sm~pill), 스타일 |

### 2. 컴포넌트 라이브러리 구축

example 파일에서 반복적으로 등장하는 UI 패턴을 추출하여 독립 컴포넌트로 만든다.

#### Primitive 컴포넌트 (기본 블록)

| 컴포넌트 | 설명 | example 내 출현 |
|----------|------|-----------------|
| `Badge` | 상태 뱃지 (Succeeded, Failed, Preset, ON 등) | 10+ |
| `StatusDot` | 컬러 원형 상태 표시기 | 5+ |
| `Button` | Primary/Secondary/Ghost/Icon 버튼 | 5+ |
| `Checkbox` | 체크박스 (테이블 행 선택용) | 다수 |
| `Divider` | 수평 구분선 | 다수 |

#### Composite 컴포넌트 (조합 블록)

| 컴포넌트 | 설명 | example 내 출현 |
|----------|------|-----------------|
| `Breadcrumb` | 경로 탐색 네비게이션 | 1 |
| `Tabs` | 탭 네비게이션 (카운트 뱃지 포함) | 1 |
| `Pagination` | 페이지네이션 (페이지 번호 + 페이지당 개수) | 2 (중복) |
| `Select` | 드롭다운 필터 | 3+ |
| `SearchInput` | 키워드 검색 필드 | 1 |
| `ContextMenu` | 우클릭 메뉴 (아이콘 + 구분선 + 링크) | 2 (중복) |
| `CollapsibleSection` | 접기/펼치기 섹션 | 4 |
| `StatusCard` | 상태 요약 카드 (아이콘 + 라벨 + 카운트) | 4 |
| `Tooltip` | 다크 배경 툴팁 | 2 |

#### Pattern 컴포넌트 (레이아웃 패턴)

| 컴포넌트 | 설명 |
|----------|------|
| `DataTable` | 체크박스 + 헤더 + 행 + 정렬 지원 테이블 |
| `PageHeader` | 브레드크럼 + 타이틀 + 액션 버튼 |

### 3. 유틸리티

| 유틸 | 설명 |
|------|------|
| `cn()` | clsx + tailwind-merge 유틸 (이미 프로젝트에 있다면 재사용) |
| `cva` | class-variance-authority로 컴포넌트 variant 관리 |

## 기술적 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| 배치 위치 | `content/ccp/_components/` | FSD shared 오염 방지, co-location, `_` prefix로 슬라이드 제외 |
| 토큰 형식 | TypeScript const 객체 | 타입 안전성, IDE 자동완성, 별도 CSS 파일 불필요 |
| 스타일링 | Tailwind CSS 유틸리티 | 프로젝트 기존 스택과 일치 |
| Variant 관리 | cva (class-variance-authority) | 타입 안전한 variant 정의, Tailwind과 궁합 좋음 |
| 아이콘 | lucide-react | 프로젝트에 이미 설치됨, SVG import 대체 |

## 예상 결과

디자인 시스템 구축 완료 후:

1. CCP 슬라이드에서 일관된 UI를 빠르게 조립할 수 있다
2. 새 화면(실행이력, 파이프라인 상세 등) 추가 시 컴포넌트 재사용으로 생산성 향상
3. 디자인 변경 시 토큰만 수정하면 전체 반영
4. example 파일의 하드코딩된 스타일이 체계적인 토큰 시스템으로 대체됨
