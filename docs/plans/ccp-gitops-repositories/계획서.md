# CCP GitOps Repositories 화면 설계서

## 목표
- CCP(CONE-Chain Platform)의 GitOps > Repositories 메뉴 전체 화면을 설계 문서로 정리
- Figma 디자인(node `3008:6878`)을 기반으로 화면 구성, 컴포넌트, 데이터 흐름을 명확히 기술
- 향후 tino 프로젝트에서 이 설계서를 기반으로 프레젠테이션/문서를 생성할 수 있도록 함

## 배경
- CCP는 Gitea + ArgoCD 기반의 GitOps 관리 플랫폼
- Repositories 메뉴는 Git 리포지토리의 연결, 목록 조회, 상세 보기/편집 기능을 제공
- Figma 파일: `iRANoeSmjtBhtTmrTHsgyf`, 섹션: `3008:6878` (GitOps > Repositories)

### 1. 리포지토리 목록 (`3008:6879`)
- **경로**: GitOps > Repositories > 리포지토리 목록
- **설명**: 등록된 모든 리포지토리를 테이블로 조회하는 메인 화면
- **주요 구성**:
  - **요약 카드 영역**: Connected / Disconnected / Unknown / Unmanaged 각 개수 표시
  - **탭 필터**: GitOps 연결 현황 (배지: 60)
  - **테이블 상단 컨트롤**:
    - Topic 탭: 전체 | Pipeline | Manifest | Artifact
    - 필터: 연결 상태(select), 이름(select), 키워드 검색(input)
    - 버튼: `+ 연결` (primary), `새로고침` (primary)
  - **테이블 컬럼**: 체크박스 | 연결(상태 dot) | Topic(배지) | 이름 | 프로젝트 | 네임스페이스 | 생성시간 | 액션(...)
  - **페이지네이션**: `<< < 1 2 3 4 5 ... 32 > >>` + 페이지 크기 선택(5)
- **연결 상태 색상**:

### 2. 리포지토리 연결 - 설정 탭 (`3008:7327`)

    - Skip server verification: toggle ON
    - Force HTTP basic auth: toggle ON
    - Enable LFS support (Git only): toggle OFF
  - 프록시: Enable OCI toggle OFF
- **하단 버튼**: 취소(gray) | 임시저장(gray) | 연결(primary blue)
- **우측 패널**: 빈 영역 (gray `#EEF0F2` 배경, 528px 너비)
