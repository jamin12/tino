# Tino - AI 기반 기획/설계 도구

## 목표

AI(Claude)가 구조화된 기획 문서를 자동 생성하고, 슬라이드(PPT처럼) + 캔버스(Figma처럼) 두 가지 모드로 보여주는 React 기반 도구를 만든다.

## 배경

- **Figma**: 디자인 특화, AI 통합 부족, 기획 문서 작성에 과도한 도구
- **PPT**: 슬라이드 기반이지만 다이어그램/와이어프레임 불편, AI 통합 불가
- **Notion/Markdown**: 텍스트 중심, 시각적 표현 제한적
- **HTML 기획서**: 코드로 작성하면 뷰잉이 PPT/Figma처럼 깔끔하지 않음

**핵심 페인 포인트**: AI가 기획서를 생성해줘도, 그걸 보기 좋게 보여주는 도구가 없다.

## 핵심 워크플로우

```
사용자가 프롬프트 입력
  → AI(Claude)가 구조화된 JSON 문서 생성
    → Tino가 프리빌트 컴포넌트로 렌더링
      → 슬라이드 모드 또는 캔버스 모드로 뷰잉
        → 사용자가 직접 수정/편집
```

## 접근 방식

### AI 렌더링: JSON Schema + 프리빌트 컴포넌트 라이브러리

AI가 TSX 코드를 직접 생성하는 대신, **구조화된 JSON을 출력**하고 미리 만들어둔 컴포넌트가 렌더링한다.

```json
{
  "type": "flowchart",
  "nodes": [
    { "id": "n1", "label": "사용자", "position": { "x": 0, "y": 0 } },
    { "id": "n2", "label": "로그인", "position": { "x": 200, "y": 0 } }
  ],
  "edges": [
    { "source": "n1", "target": "n2", "label": "접속" }
  ]
}
```

→ 이 JSON을 `<Flowchart>` 컴포넌트가 받아서 렌더링

### 왜 이 방식인가

| 장점 | 설명 |
|------|------|
| 안전 | eval이나 런타임 컴파일 불필요 |
| 빠름 | JSON 파싱만으로 즉시 렌더링 |
| 예측 가능 | AI 출력이 항상 유효한 컴포넌트로 매핑 |
| 편집 용이 | JSON 구조를 UI에서 직접 수정 가능 |
| 확장성 | 새 컴포넌트 타입 추가하면 AI도 바로 사용 가능 |

## 기술적 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| AI 렌더링 방식 | JSON Schema + 프리빌트 컴포넌트 | 안전, 빠름, 편집 용이 |
| 캔버스 모드 | tldraw (Phase 2) | React 네이티브, 커스텀 셰이프, 줌/팬 내장 |
| 다이어그램 | @xyflow/react (React Flow v12) | 노드 기반 UI 최강, 커스텀 노드/엣지 |
| 로컬 DB | Dexie.js (IndexedDB) | 대용량, 구조화 쿼리, 오프라인 |
| 스키마 검증 | Zod | AI 출력 유효성 검증 |
| 코드 하이라이팅 | Shiki | 빠름, 다양한 테마 |
| 마크다운 렌더링 | react-markdown | 경량, React 통합 |
| 자동 레이아웃 | dagre | 플로우차트 노드 자동 배치 |

## 기능 목록

## 화면 구성

```
/                      → 홈 (문서 목록, 카드 그리드)
/new                   → 새 문서 (AI 프롬프트 입력)
/document/:id          → 문서 뷰/편집 (슬라이드 or 캔버스 모드)
/document/:id/present  → 프레젠테이션 모드 (전체화면)
```

## 영향 범위

기존 FSD 스캐폴딩 위에 새 모듈 추가. 기존 파일 수정 최소화.

### 신규 생성
- `entities/document/` - 문서 도메인 (타입, API, 쿼리)
- `entities/element/` - 엘리먼트 렌더러 레지스트리
- `entities/ai/` - Claude API 클라이언트
- `features/create-document/` - 문서 생성
- `features/generate-document/` - AI 생성
- `features/slide-viewer/` - 슬라이드 네비게이션
- `widgets/document-list/` - 문서 목록 UI
- `widgets/slide-presenter/` - 슬라이드 뷰어 UI
- `widgets/element-renderer/` - 엘리먼트 렌더링 디스패처
- `shared/ui/components/` - 프리빌트 컴포넌트 (text, diagram)
- `shared/ui/layouts/` - 앱 레이아웃
- `shared/lib/db/` - Dexie.js 인스턴스
- `pages/document/`, `pages/new/`, `pages/present/`

### 수정
- `src/app/router.tsx` - 라우트 추가
- `apps/web/package.json` - 패키지 추가

## 예상 결과

MVP 완료 후:
1. 앱을 열면 문서 목록이 보인다
2. "새 문서" 클릭 → 프롬프트 입력 → AI가 기획서 생성
3. 생성된 문서를 슬라이드로 넘기며 볼 수 있다
4. 텍스트, 플로우차트, 마인드맵이 깔끔하게 렌더링된다
5. 전체화면 프레젠테이션 모드로 발표할 수 있다
6. 문서가 로컬(IndexedDB)에 자동 저장된다
