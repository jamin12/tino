# 맥락 노트: Tino - AI 기반 기획/설계 도구

## 현재 코드베이스 상태

pnpm 워크스페이스 모노레포. FSD 아키텍처로 스캐폴딩 완료 상태. 실제 비즈니스 로직은 아직 없음.

- `apps/web/` - React 19 + Vite 7 SPA (FSD 구조)
- `packages/shared/` - @workpulse/shared (공통 타입, 유틸)
- 기존 기술 스택: React 19, TanStack Query 5, Zustand 5, Axios, Tailwind CSS 4, TypeScript

## 제약 조건

### 기술적 제약
- **FSD 규칙 준수**: 단방향 의존, 동일 레이어 cross-import 금지, index.ts 캡슐화
- **Path Alias**: @app, @pages, @widgets, @features, @entities, @shared 사용 필수
- **Public API**: 외부에서는 index.ts를 통해서만 import
- **Node.js 버전**: 22.10.0 (Vite 7이 22.12+ 권장하지만 동작함)

### 비즈니스 제약
- **개인 프로젝트**: 협업 기능 불필요
- **로컬 우선**: 서버 없이 브라우저만으로 동작
- **AI API 키**: 사용자가 직접 입력 (로컬 저장)

## 주의사항

- **FSD cross-import 금지**: `features/create-document`에서 `features/generate-document` 직접 import 금지. 두 기능 결합이 필요하면 `widgets`에서 조합.
- **AI API 키 보안**: localStorage에 저장 시 XSS 취약점 주의. 최소한 난독화 권장.
- **AI 출력 검증**: Claude가 항상 유효한 JSON을 생성하진 않음. Zod 검증 + fallback 필수.
- **tldraw 번들 크기**: 상당히 큼. Phase 2에서 lazy loading 필수.
- **React Flow 읽기 모드**: 슬라이드 뷰에서는 인터랙션 비활성화 필요 (nodesDraggable=false 등).
