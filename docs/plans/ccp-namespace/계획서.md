# CCP Namespace 관리 기획

## 목표
- CCP 대시보드에 **Namespace 관리** 화면을 추가하여 Kubernetes 네임스페이스의 생성/조회/삭제를 UI 목업으로 설계한다
- 실제 동작: Gitea API를 통해 두 개 레포(k8s-cicd-init, k8s-app-init)에 자동으로 폴더/파일을 생성/삭제한다

## 배경

### NS 생성 시 벌어지는 일

#### 작업 1: k8s-cicd-init 레포
- `sample-cicd/` 폴더를 복제하여 `{ns}-cicd/` 생성
- 내부 모든 파일에서 `sample` → `{ns}` 치환
- 생성되는 파일:
  - `{ns}-cicd/namespace.yaml` — NS 정의 (name, labels)
  - `{ns}-cicd/kustomization.yaml` — Kustomize 패치 (PV, PVC, RoleBinding 등)
- 루트 `kustomization.yaml`의 resources에 `{ns}-cicd` 자동 추가
- `clusterrolebinding.yaml`에 새 NS의 ServiceAccount ClusterRoleBinding 추가

#### 작업 2: k8s-app-init 레포
- 사용자가 선택한 환경(dev, prd, stg 등)별로:
  - `overlays/{env}/sample-{env}/` 복제 → `overlays/{env}/{ns}-{env}/` 생성
  - 내부 파일에서 `sample` → `{ns}` 치환
  - `overlays/{env}/kustomization.yaml`의 resources에 `{ns}-{env}` 추가
- **(추후 처리) istio-system VirtualService**:
  - `overlays/{env}/istio-system/sample-vs.yaml` 복제 → `{ns}-vs.yaml`
  - 내부 호스트명, 서비스명에서 `sample` → `{ns}` 치환
  - `overlays/{env}/istio-system/kustomization.yaml`의 resources에 추가
  - ※ 이번 기획에서는 슬라이드에 포함하지 않음, 추후 별도 처리
- 환경 관리:
  - 기존 환경(dev, prd, stg) 선택 가능
  - 새로운 커스텀 환경 생성 가능

### NS 삭제
- **환경별 삭제**: 특정 환경(예: dev만)의 관련 파일/참조 제거
- **전체 삭제**: k8s-cicd-init의 `{ns}-cicd/` + k8s-app-init의 모든 환경 폴더 + (추후) VirtualService + 참조 일괄 제거
- NS 이름 변경은 불가 (삭제 후 재생성으로 대응)

## 화면 구성

### Screen 1: Namespace 목록 (Slide01NamespaceList)
- **screenId**: `CCP-NS-001`
- 전체 NS 목록을 테이블로 표시
- 컬럼: GitOps 상태, NS 이름, 연결된 환경(뱃지), CICD 상태, 생성일
- StatusSummary: 전체/활성/비활성 통계
- 상단 필터: 검색, 환경 필터
- ActionMenu: 상세보기, 환경 추가, 환경별 삭제, 전체 삭제
- 생성 버튼 → NS 생성 화면

### Screen 2: Namespace 생성 (SlideNamespaceCreate)
- **screenId**: `CCP-NS-002`
- 탭: 기본정보 | 환경설정
- 기본정보 탭:
  - NS 이름 입력 (필수, 영소문자+하이픈, 자동으로 `-cicd` 접미사 표시)
  - 설명 입력
  - Labels/Annotations (KeyValueEditor)
- FormBanner: "네임스페이스를 생성하면 CICD 파이프라인과 배포 환경이 자동으로 구성됩니다."

### Screen 3: Namespace 생성 - 환경설정 탭 (SlideNamespaceCreateEnv)
- **screenId**: `CCP-NS-003`
- 환경설정 탭 활성 상태
- 기존 환경 목록 (dev, prd, stg) 체크박스로 선택
- "새 환경 추가" 버튼 → 환경 이름 입력 필드 표시
- 선택된 환경별 미리보기 (어떤 리소스가 생성되는지)
- ~~VirtualService 호스트 패턴: `{env}-{ns}.cone-chain.com`~~ (추후 처리)

### Screen 4: Namespace 상세 (SlideNamespaceDetail)
- **screenId**: `CCP-NS-004`
- ListDetailLayout: 좌측 NS 목록 + 우측 상세 패널
- 상세 패널:
  - ResourceDetailPanel (NS 이름, 상태)
  - IconNav: 요약/환경/YAML
  - 요약 탭: 기본 정보 (이름, 생성일, 라벨 등)
  - InfoRow로 key-value 표시

### Screen 5: Namespace 상세 - 환경 목록 (SlideNamespaceDetailEnvs)
- **screenId**: `CCP-NS-005`
- 상세 패널의 "환경" 탭 활성
- 연결된 환경 목록을 카드/테이블로 표시
  - 환경명, 상태, 생성일
- 환경 추가 버튼
- 환경별 삭제 버튼 (각 행에 삭제 ActionMenu)

### Screen 6: Namespace 환경 추가 (SlideNamespaceEnvAdd)
- **screenId**: `CCP-NS-006`
- 기존 NS에 새 배포 환경을 추가하는 화면
- 기존 환경 선택 또는 새 환경 이름 입력
- 생성될 리소스 미리보기

### Screen 7: Namespace 삭제 확인 (SlideNamespaceDelete)
- **screenId**: `CCP-NS-007`
- 삭제 유형 선택:
  - 환경별 삭제: 드롭다운/체크박스로 삭제할 환경 선택
  - 전체 삭제: CICD + 모든 환경 일괄 삭제
- 삭제 영향 범위 표시 (어떤 리소스가 제거되는지)
- 확인 입력 (NS 이름 타이핑)

## 예상 결과
- `apps/web/src/content/ccp/cicd/namespace/` 디렉토리에 7개 슬라이드 파일 생성
- 기존 CCP storage 슬라이드와 동일한 패턴, 동일한 _components 사용
- 화면 간 links로 네비게이션 관계 정의
