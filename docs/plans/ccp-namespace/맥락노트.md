## 제약 조건

### 기술적 제약
- 슬라이드 파일에서 raw HTML 태그(div, span 등) 사용 금지 → _components만 사용
- DataTable render 함수 내부도 컴포넌트만 (TextCell, CellGroup, Badge 등)
- 모든 import는 `../../_components`에서만
- slideMeta 필수: screenId, title, section, links

### 패턴 준수
- 목록 화면: CcpDashboardLayout > ContentSection > StatusSummary + FilterBar + DataTable + Overlay(ActionMenu) + Pagination
- 생성 화면: CcpDashboardLayout > ContentSection > FormBanner + Tabs + 폼필드 + FormActions
- 상세 화면: CcpDashboardLayout > ListDetailLayout > ResourceDetailPanel + IconNav + InfoRow

## 핵심 결정 로그

| 시점 | 결정 | 이유 | 대안 |
|------|------|------|------|
| 초기 | _security 보안정책 제외 | 사용자 요청: 이번 기획에서 제외 | 추후 별도 기획 |
| 초기 | kustomization.yaml 자동 업데이트 | 내부 처리로 결정 | UI에서 수동 편집 |
| 초기 | NS 이름 변경 불가 | 사용자 결정 | 삭제 후 재생성 |
| 초기 | 삭제는 환경별 + 전체 두 가지 | 유연한 운영 필요 | 전체 삭제만 |
| 초기 | VirtualService 추후 처리 | 사용자 요청: 이번 기획에서는 슬라이드 미포함, 계획서에 기록만 | 함께 생성 |
| 초기 | ClusterRoleBinding 자동 추가 | CICD 파이프라인 권한 필요 | 수동 추가 |

## 주의사항
- sideMenuItems에 "네임스페이스" 항목 추가 필요 (storage 슬라이드의 사이드메뉴 참고)
- screenId 네이밍: `CCP-NS-001` ~ `CCP-NS-007`
- section: "CI/CD 네임스페이스"
- 환경 목록은 하드코딩이 아니라 동적 (기존 환경 + 새 환경 생성 가능)
