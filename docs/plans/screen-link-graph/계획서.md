# 화면 연동 정보 & 그래프 시각화

## 목표
- 각 슬라이드(화면)가 어떤 다른 화면과 연동되는지 메타데이터로 정의
- 특정 화면을 선택하면 앞/뒤로 연결된 화면들을 **그래프 형식**으로 시각화

## 배경
- 현재 CCP 프로젝트에 37개 슬라이드가 존재하지만, 화면 간 관계(목록→상세, 목록→생성 등)가 코드 안에 암묵적으로만 존재
- `SlideMeta`에 `screenId`가 있지만, 상세/생성 슬라이드 22개는 `screenId`가 없음
- 화면 간 연결 관계(links)가 메타데이터로 정의되어 있지 않음

## 접근 방식

### Phase 1: 연동 메타데이터 정의
1. `SlideMeta` 타입에 `links` 필드 추가
2. 모든 슬라이드에 `screenId` 부여 (누락된 22개)
3. 각 슬라이드의 `slideMeta`에 `links` 배열 추가
4. content-api에서 링크 정보를 수집하는 유틸 함수 제공

### Phase 2: 그래프 시각화 UI
1. 문서 페이지에서 특정 슬라이드 선택 시 연결된 화면을 그래프로 표시
2. React Flow 기반 방향성 그래프 렌더링
3. 현재 화면을 중심으로 1-depth 연결 화면을 시각화
4. 노드 클릭 시 해당 슬라이드로 이동

## 기술적 결정 사항

| 결정 | 선택 | 이유 |
|------|------|------|
| 링크 데이터 위치 | `SlideMeta.links` | 슬라이드 파일과 함께 관리되어 응집도 높음 |
| 링크 타입 | `navigate`, `modal`, `tab`, `action` | CCP 화면 간 전환 패턴을 모두 커버 |
| screenId 네이밍 | 기존 패턴 유지 (CCP-{섹션}-{번호}) | 일관성 |
| 그래프 라이브러리 | React Flow (이미 프로젝트에 있음) | Flowchart 컴포넌트에서 이미 사용 중 |
| 그래프 배치 | 선택한 화면 중심 방사형 | 앞/뒤 관계를 직관적으로 표현 |

## 영향 범위

### Phase 1 (연동 메타데이터)
| 파일 | 변경 내용 |
|------|-----------|
| `entities/document/types/document.ts` | `ScreenLink` 타입 추가, `SlideMeta.links` 필드 추가 |
| `entities/document/api/content-api.ts` | 링크 맵 수집 유틸 추가 |
| `content/ccp/**/Slide*.tsx` (37개) | `screenId` 누락분 추가 + `links` 배열 추가 |

### Phase 2 (그래프 시각화)
| 파일 | 변경 내용 |
|------|-----------|
| `features/screen-graph/` | 새 feature: 그래프 상태 관리 및 데이터 변환 |
| `widgets/screen-graph-viewer/` | 새 widget: React Flow 기반 그래프 UI |
| `pages/document/DocumentPage.tsx` | 그래프 뷰어 통합 |

## 예상 결과
- Phase 1 완료 시: 모든 슬라이드가 `screenId`와 `links`를 가지며, API를 통해 전체 화면 연결 맵을 조회 가능
- Phase 2 완료 시: 문서 뷰어에서 슬라이드 선택 시 연결된 화면들이 인터랙티브 그래프로 표시
