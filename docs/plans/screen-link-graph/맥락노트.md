## ScreenLink 타입 설계

```typescript
interface ScreenLink {
  /** 연결 대상 화면의 screenId */
  targetScreenId: string;
  /** 연결 유형 */
  type: "navigate" | "modal" | "tab" | "action";
  /** 연결 설명 (예: "행 클릭 시 상세 이동") */
  label?: string;
}
```

**linkType 정의:**
| type | 의미 | 예시 |
|------|------|------|
| `navigate` | 화면 전환 (페이지 이동) | 목록 → 상세, 사이드메뉴 클릭 |
| `modal` | 모달/다이얼로그 오픈 | 생성 버튼 → 생성 다이얼로그 |
| `tab` | 같은 화면 내 탭 전환 | 상세 화면 탭 이동 |
| `action` | 상태 변경/트리거 | 저장 → 목록 복귀, 삭제 → 목록 복귀 |

## CCP 화면 연결 관계 분석

### 공통 패턴
1. **사이드메뉴 → 목록**: 대시보드, StorageClasses, PV, PVC, ConfigMaps, Secrets, 리포지토리
2. **목록 → 상세**: 테이블 행 클릭 → 상세 첫 탭
3. **목록 → 생성**: "생성" 버튼 → 생성 다이얼로그
4. **상세 탭 ↔ 상세 탭**: 같은 리소스의 탭 간 전환
5. **생성 탭 ↔ 생성 탭**: 생성 플로우 탭 간 전환
6. **생성 → 목록**: 생성 완료/취소 시 목록 복귀

### 구체적 연결 맵
```
대시보드(DSH-001) ←→ 위젯편집(DSH-002)
대시보드(DSH-001) → StorageClasses(STR-001), PV(STR-002), PVC(STR-003), ...

StorageClasses 목록(STR-001)
  → 상세: 프로비저너(STR-001-D1), Parameters(STR-001-D2)
  → 생성: 공통(STR-C00) → Provisioner(STR-001-C2) → Parameters(STR-001-C3)

PV 목록(STR-002)
  → 상세: Spec(STR-002-D1), PVC바인딩(STR-002-D2)
  → 생성: 공통(STR-C00) → Spec(STR-002-C2)
  → 관리모달(STR-002-M)

PVC 목록(STR-003)
  → 상세: Spec(STR-003-D1), Status(STR-003-D2)
  → 생성: 공통(STR-C00) → Spec(STR-003-C2)

ConfigMaps 목록(STR-004)
  → 상세: 요약(STR-004-D1), 기본정보(STR-004-D2), Data(STR-004-D3), 설정(STR-004-D4), YAML(STR-004-D5)
  → 생성: 공통(STR-C00) → Data(STR-004-C1)

Secrets 목록(STR-005)
  → 상세: 타입(STR-005-D1), Data(STR-005-D2)
  → 생성: 공통(STR-C00) → Type(STR-005-C1) → Data(STR-005-C2)

리포지토리 목록(GIT-001)
  → 연결: 기본정보(GIT-002) → GitOps(GIT-003)
  → 상세: 기본정보(GIT-004), 코드(GIT-005), 편집(GIT-006)
```

## 주의사항
- `SlideStorageCreate.tsx`는 5가지 리소스 타입이 공유하는 공통 생성 1단계 → screenId를 `CCP-STR-C00`으로 부여
- 상세 탭 간 연결은 `tab` 타입으로, 목록→상세는 `navigate` 타입으로 구분
- links는 **단방향** (출발 화면에서만 선언) → content-api에서 역방향 맵도 자동 생성
- Phase 2의 그래프에서는 양방향을 모두 표시
